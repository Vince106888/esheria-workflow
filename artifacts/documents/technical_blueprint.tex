\sectionbanner{01}{System Architecture}{Five-layer stack — control plane, data plane, and experience surfaces}{Teal}

\subsection{Architecture Overview}

The platform is structured as five horizontal layers. Each layer has a bounded responsibility
and communicates only with adjacent layers. The canonical architecture fact: \textbf{no layer
has a direct dependency on a non-adjacent layer.}

\medskip
\rowcolors{2}{Light}{White}
\begin{tabularx}{\linewidth}{%
  C{11mm}%
  >{\headingfont\bfseries\small\color{Text}}L{28mm}%
  L{38mm}%
  >{\small\color{TextLt}}X%
}
\rowcolor{Navy2}
\thead{Layer} & \thead{Name} & \thead{Canonical Prefix} & \thead{Responsibility} \\
\midrule
\tmono{L5} & Experience
  & \code{/} (frontend routing)
  & Institution Workspace, Operator Console, Public surface. Never calls \code{/internal/*}. \\
\tmono{L4} & Control Plane
  & \code{/api/workflows/*}
  & Workflow Engine, State Machine, Approval Router, Escalation Manager, Audit/Event Log. \\
\tmono{L3} & Object Model
  & \code{schema.*}
  & Nine canonical entities shared across all layers. Schema is locked in P1 before service code. \\
\tmono{L2} & Connectors
  & \code{/internal/*}
  & Connector Gateway mediating all platform-to-service calls. Owns retry, auth, error normalisation. \\
\tmono{L1} & Existing Services
  & \code{(service-native)}
  & Current repos: \code{esheria-ocr}, \code{contract-engine}, \code{esheria-ai}, \code{lexdraft}, \code{esheria-analytics}. Unchanged. \\
\bottomrule
\end{tabularx}

\medskip

\begin{decisionbox}{// CANONICAL ARCHITECTURE DECISION}
The Canonical Object Model (L3) must be defined and locked in Phase~1 before any service code
is written in Phase~2. Schema changes within a major version are treated as public API breaking
changes and require migration plans, backward-compat periods, and explicit engineering approval.
\end{decisionbox}

\subsection{Control Plane vs Data Plane}

\begin{itemize}
  \item \textbf{Control Plane (L4):} Governs workflow lifecycle — creates \code{WorkflowRun} objects,
    routes steps to operators, enforces SLAs, manages approval chains, and writes every state
    transition to the immutable audit log.
  \item \textbf{Data Plane (L2+L1):} Executes capability operations — OCR, extraction, compliance
    check, research, drafting. Reads and writes canonical objects at L3. Never controls workflow
    routing decisions.
  \item \textbf{Boundary rule:} A capability service \emph{never} triggers workflow transitions.
    It returns results to the Control Plane, which decides the next step.
\end{itemize}

\subsection{Relationship Rules}

\begin{itemize}
  \item Every entity except \entity{Organization} carries an \code{org\_id} foreign key. Row-level
    security enforces org-scoped isolation at the database layer.
  \item \entity{Matter} is the primary grouping unit. All documents, workflow runs, and reviews
    attach to a Matter.
  \item \entity{Clause} and \entity{Risk} are write-once after extraction. Risk findings are
    appended, never mutated — only resolved through a \entity{Review} decision.
  \item \entity{Action} records are immutable once written. They form the audit trail for a
    \entity{WorkflowRun}.
  \item A \entity{Citation} (not listed above as a top-level entity) is an evidence record linking
    an \entity{Action} to a source document — used by the Research and Compliance operators.
\end{itemize}

\sectionbanner{04}{Workflow Engine Design}{State machine runtime with deterministic transitions and full audit}{Teal}

\subsection{Design Principles}

\begin{itemize}
  \item \textbf{Deterministic transitions:} Every step has a single defined next-state for each
    possible outcome. No implicit routing logic.
  \item \textbf{Idempotent operators:} Each operator step can be safely retried without side
    effects — output objects are upserted by \code{(run\_id, step\_id)}.
  \item \textbf{Immutable audit log:} Every state transition writes an append-only record to the
    event log. Log records are never updated or deleted.
  \item \textbf{Human-in-loop:} The Approval Router is a first-class step, not an afterthought.
    Workflows pause in \code{AWAITING\_REVIEW} until a human decision is received.
  \item \textbf{SLA enforcement:} The Escalation Manager polls pending reviews and fires escalation
    events at configured deadline thresholds (typically 80\% and 100\% of SLA window).
\end{itemize}

\subsection{Tenancy Architecture}

The platform uses a \textbf{logical multi-tenancy model} with shared infrastructure and database-layer
isolation. Org-scoped row-level security (RLS) on PostgreSQL enforces data isolation at the query
layer — no application-level filtering is trusted as the primary isolation mechanism.

Every authenticated request carries a JWT containing:

  "scopes": ["matters:read", "documents:write", "workflows:trigger", ...],
  "exp":    <unix\_timestamp>,
  "iss":    "esheria-auth"
\}
\end{codebox}

\subsection{Connector Contract Specification}

Each connector in L2 must implement the following interface contract:

\medskip
\begin{codebox}
interface Connector {
  id:            string;           // e.g. "ocr-connector"
  targetService: string;           // e.g. "esheria-ocr"
  status:        "active" | "planned" | "future";

  // Called by Connector Gateway for every request
  call(endpoint: string, payload: object): Promise<ConnectorResult>;

  // Error normalisation — maps service errors to platform error codes
  normalise(error: ServiceError): PlatformError;

  // Health check — called by monitoring every 30s
  health(): Promise<{ ok: boolean; latencyMs: number }>;
}

interface ConnectorResult {
  success:  boolean;
  data:     object;
  metadata: { serviceVersion: string; latencyMs: number };
}
\end{codebox}

\subsection{Gateway Behaviour}

\begin{itemize}
  \item \textbf{Retry policy:} Exponential backoff, max 3 retries, 500ms base delay.
    \code{4xx} errors are \emph{not} retried (client error). \code{5xx} and timeouts are retried.
  \item \textbf{Circuit breaker:} Opens after 5 consecutive failures within a 30-second window.
    Half-open probe every 10s. Failing-open is never permitted — the workflow step fails cleanly.
  \item \textbf{Timeout budget:} Per-connector timeout configured in registry.
    Default 10s. OCR has 60s budget. Research has 30s budget.
  \item \textbf{Error normalisation:} Every connector maps its service's error taxonomy to the
    platform's 6-class error model: \code{AUTH\_ERROR}, \code{VALIDATION\_ERROR},
    \code{SERVICE\_UNAVAILABLE}, \code{RATE\_LIMITED}, \code{TIMEOUT}, \code{INTERNAL\_ERROR}.
\end{itemize}

Build in this order: \code{ocr-connector} first (simplest, no ML dependency, validates
gateway plumbing), then \code{contract-connector} (highest business value in MVP),
then \code{research-connector} (needed for citation evidence in Review briefing).
Stub \code{drafting-connector} and \code{analytics-connector} in P2 to unblock demo.
\end{decisionbox}

  \item \textbf{Connector Gateway skeleton:} Auth passthrough, retry/backoff, error normalisation.

  \item \textbf{OCR connector:} First real connector. Validates the gateway contract end-to-end.
  \item \textbf{Contract connector:} Extraction and risk scoring. Unlocks the core value step.
  \item \textbf{Research connector:} Citation retrieval. Completes the evidence chain.
  \item \textbf{Reviewer dashboard v1:} Minimal UI — matter list, document viewer, risk findings,
    approve/reject action. Authentication via institution workspace auth flow.
  \item \textbf{End-to-end demo flow:} Single contract through all 10 steps. Audit log visible.
    Human reviewer makes a decision. Analytics event emitted (stubbed).
\end{enumerate}
